TxArguments

A TxArguments is used to pass arguments to ITxPlugin. The keys and values are both stored as String. Accessor methods, such as GetValue<T>(String, TxArguments.TryParseFunc<T>, T) and TryGetValue<T>(String, out T, TxArguments.TryParseFunc<T>) are provided to extract and parse values to any type. Helper methods are provided for accessing primitive types (IsPrimitive) and String. A TxArguments is immutable.

TxArguments.BadLookupException

An exception that is thrown when a lookup within TxArguments fails. For example, when GetValue<T>(String, TxArguments.TryParseFunc<T>) is called with a non-existent key, or parsing the associated value fails.

TxBitmap

Helpers for image bitmaps. Here are some concepts that you should be familiar with:

  • Sampling factor: The width (or height) of a square grid of pixels from a source SKBitmap that an ITxTextBitmapGenerator maps to exactly one Char in ITxTextBitmap. That is, an ITxTextBitmapGenerator, generator, using a sampling factor, factor, will produce an ITxTextBitmap, textBitmap from an SKBitmap, bitmap, such that textBitmap.Width * factor == bitmap.Width and textBitmap.Height * factor == bitmap.Height. For example, a sampling factor of 3 means that a 3x3 square (of 9 pixels) in a bitmap will be used to come up with exactly one output Char. Consequently, a sampling factor of 1 maps every pixel to one Char and the size of the generated textBitmap and source bitmap are equal.

  • Perfect sampling factor: A sampling factor, factor that can break up a whole SKBitmap completely into a grid of squares of with side length of factor. That is, both the bitmap's height and width is divisible by factor. A perfect sampling factor ensures that the same number of pixels are used to determine every Char in ITxTextBitmap. A perfect sampling factor also implies that the output image maintains the source aspect ratio without cropping the image. There is always at least one perfect sampling factor, 1.

TxPluginAttribute

A TxPluginAttribute "exports" the ITxPlugin type from a prebuilt assembly. Texart will automatically search the assembly for this attribute and instantiate an instance. This attribute can only be applied to classes. The applied class MUST implement ITxPlugin and MUST be have a public constructor that takes no arguments.

TxPluginResourceLocator

A domain-specific URI-like string that is used to identity a plugin assembly and objects inside it. The properties and formatting is a strict subset of a URI and thus all TxPluginResourceLocators are valid (absolute) URIs. Relative resource URIs can be represented by TxPluginResourceLocator.RelativeLocator instead.

https://tools.ietf.org/html/rfc3986#section-3 defines the syntax of URIs:

//    foo://example.com:8042/over/there?name=ferret#nose
//    \_/   \______________/\_________/ \_________/ \__/
//     |           |            |            |        |
//  scheme     authority       path        query   fragment
//     |   _____________________|__
//    / \ /                        \
//    urn:example:animal:ferret:nose

See https://en.wikipedia.org/wiki/Uniform_Resource_Identifier#/media/File:URI_syntax_diagram.png for a simple diagram.

The anatomy of a TxPluginResourceLocator looks like:

//  Scheme       AssemblyPath       ResourcePath
//    |      _________|________   _______|_____
//   / \    /                  \ /             \
//   foo:///path/to/assembly.dll:path/to/resource
//       \_/                    |
//        |                     |
// (empty Authority)  AssemblyResourceSeparator (:)

From the above, only the following properties are allowed:

  • scheme: Scheme.
  • authority: This is required for a URI but in our case, this MUST be empty.
  • path: The path MUST contain at least one segment containing AssemblyResourceSeparator, the last of which is used to partition the path into AssemblyPath and ResourcePath. ResourcePath is allowed to be empty. These restrictions may be relaxed in the future, but will always be compliant with the latest URI RFC. Note that the current restrictions imply that http or https schemes cannot be used, since they require a hostname.

TxPluginResourceLocator.FormatException

A TxPluginResourceLocator.FormatException is thrown when an attempt is made to create TxPluginResourceLocator or TxPluginResourceLocator.RelativeLocator with an invalid string.

TxPluginResourceLocator.RelativeLocator

A TxPluginResourceLocator.RelativeLocator represents the ResourcePath part of TxPluginResourceLocator. It represents a resource relative to AssemblyPath.

TxReferenceScheme

A TxReferenceScheme represents a URI scheme (such as file: and https:). As such, it is also case-insensitive.

Note that not every valid URI scheme is a valid TxReferenceScheme. Specifically, the characters, +, and ., are not allowed. However, - is allowed. This restriction may be relaxed in the future.

Refer to https://tools.ietf.org/html/rfc3986#section-3.1 for information on URI schemes (which are valid TxReferenceSchemes, except for the restrictions listed above).

//    scheme      = ALPHA *( ALPHA / DIGIT / "+" / "-" / "." )

Use cases:

  • As a way to allow custom resolution strategies for #load and #r directives in Texart scripts.
  • As a part of locating resources in a ITxPlugin assembly. See TxPluginResourceLocator.

TxReferenceScheme.FormatException

A TxReferenceScheme.FormatException is thrown when an attempt is made to create TxReferenceScheme with an invalid string.

Interfaces

ITxPlugin

A plugin represents a collection of named ITxTextBitmapGenerator and ITxTextBitmapRenderer. The plugin abstraction allows sharing these types across assembly boundaries.

ITxPlugin unifies sharing "resources" (like algorithms) in the following contexts:

  • Within Texart code.
  • Within a separately compiled assembly, built against Texart.Api.
  • Within a .tx.csx script file that will be interpreted by the Texart runtime.

ITxTextBitmap

An ITxTextBitmap is a bitmap of character data – like char[,].

The API exposes an indexer-like method, char ChatAt(int x, int y) for 2D-array-like access. The internal representation is implementation-defined. The underlying computation can be performed on the fly, or wrap around a pre-populated char[,]. etc.

Implementations are strongly encouraged to be immutable.

ITxTextBitmapGenerator

An ITxTextBitmapGenerator is used to generate ITxTextBitmaps given some configuration options.

ITxTextBitmapRenderer

An ITxTextBitmapRenderer writes some text data into an output stream in an implementation-defined format.

TxArguments.IExtractable

An TxArguments.IExtractable is an object that is able to extract values out of a TxArguments instance.

This API requires the implementing class to be mutable in order to (meaningfully) extract values from arguments. Caution should be exercised if the extracted object needs to be stored beyond the scope of its declaration. If the extracted values need to be persisted, it is recommended to place them into a different (but very similar) type to this which IS immutable.
